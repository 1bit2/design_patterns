# Creational Patterns / 创建型模式

创建型模式关注**对象怎么"生出来"**，将对象的创建逻辑与使用逻辑分离，
让系统不依赖具体类的实例化方式。

## 包含的模式

| 模式 | 说明 | 状态 |
|------|------|------|
| [建造者 (Builder)](builder/) | 一步步组装复杂对象，构建与表示分离 | ✅ 已完成 |
| 工厂方法 (Factory Method) | 一个方法决定创建哪种产品 | 待补充 |
| 抽象工厂 (Abstract Factory) | 一个工厂生产一整套配套产品 | 待补充 |
| 单例 (Singleton) | 全局只有一个实例 | 待补充 |
| 原型 (Prototype) | 通过克隆已有对象创建新对象 | 待补充 |

## 核心思想

创建型模式的共同特点是将"创建什么"和"怎么创建"封装起来，
使客户端代码不直接依赖 `new`，从而获得更大的灵活性：

- **何时创建** → 延迟到运行时决定
- **创建哪个** → 通过配置或策略选择
- **怎么创建** → 隐藏复杂的初始化过程
