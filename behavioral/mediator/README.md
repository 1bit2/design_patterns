# Mediator Pattern / 中介者模式

**分类**: 行为型模式 (Behavioral)

## 解决的问题

当多个对象之间存在复杂的网状交互关系时，每个对象都需要引用
其他所有对象，导致系统难以理解和维护。

中介者模式引入一个**中介者对象**集中管理所有交互逻辑，
把多对多的网状通信变成星形结构（所有对象只与中介者通信）。

## 示例说明

本目录中的 `mediator.cpp` 以**聊天室**为例：

- `ChatRoom` — 中介者（Mediator），管理用户列表和消息路由
- `User` — 同事类（Colleague），只持有聊天室引用，不直接引用其他用户
- 支持广播消息和私聊消息
- 用户之间互不知道对方的存在，所有通信通过 ChatRoom 转发

## 编译运行

```bash
g++ -std=c++17 -o mediator mediator.cpp && ./mediator
```

## 要点

- 对象间不直接引用 → 新增/删除用户不影响其他用户
- 交互逻辑集中在中介者 → 修改路由规则只改一处
- 典型应用：聊天室、机场塔台、表单控件联动

## 注意

中介者模式的风险是中介者本身可能变得过于复杂。
如果中介者承担了过多职责，应考虑进一步拆分。
