# Structural Patterns / 结构型模式

结构型模式关注**对象怎么"拼在一起"**，通过组合、聚合、包装等方式，
把类和对象组装成更大的结构，同时保持灵活。

## 包含的模式

| 模式 | 说明 | 状态 |
|------|------|------|
| [组合 (Composite)](composite/) | 树形结构，叶子和容器统一接口 | ✅ 已完成 |
| [桥接 (Bridge)](bridge/) | 拆分抽象和实现，独立变化 | ✅ 已完成 |
| 适配器 (Adapter) | 接口转换，让不兼容的类能协作 | 待补充 |
| 装饰器 (Decorator) | 动态给对象加功能，替代继承 | 待补充 |
| 外观 (Facade) | 给复杂子系统一个简单入口 | 待补充 |
| 享元 (Flyweight) | 共享细粒度对象，节省内存 | 待补充 |
| 代理 (Proxy) | 控制对目标对象的访问 | 待补充 |

## 核心思想

结构型模式的共同特点是利用**组合/聚合**替代深层继承，
让系统在运行时可以灵活地改变对象的结构和关系：

- **组合替代继承** → 运行时动态组合，而非编译期固定
- **接口适配** → 让不兼容的组件能协同工作
- **简化访问** → 为复杂结构提供统一的访问入口
